<!-- File debug-az.prompt.md -->
You are an expert debugger, specialized in debugging complex programming issues.

You are particularly great at debugging this project, which architecture and quirks can be consulted in the [architecture document](./docs/architecture.md).

Follow these steps:

1. Review the [error logs](./logs/error.log) and identify the root cause.

2. Use the `azmcp-monitor-log-query` MCP tool to retrieve infrastructure logs from Azure.

3. Once you find the root cause, think about 3 potential solutions with trade-offs

4. Present your root cause analysis and suggested solutions with trade-offs to the user and seek validation before proceeding with fixes – do not change any files.

Repeatable Patterns

Agentic Primitives

Copilot-Instructions.md

  • Always included in every chat request.
  • For broad, workspace-wide coding standards.
  • Works across multiple editors (VS Code, Visual Studio, GitHub.com)

.instructions.md

  • Task or context-specific instructions that can target specific files or patterns.
  • Auto/manual inclusion
  • Provides more granular, task-specific instructions

.prompt.md

  • Used for fine tuning responses
  • Customizes how Copilot generates code in specific contexts.
  • Creates reusable prompts for common tasks

.agent.md

  • Sets persona
  • Configure which tools the AI can access
  • Defines behaviour, sets tone and best practices

Repeatable Patterns

Implementing complex tasks

Create explicit implementation pipelines / workflows

1

Plan

Break the task into discrete, scoped steps. Define acceptance criteria before writing code.

2

Spec

Create spec files that capture context, constraints, and expected interfaces for each component.

3

Implement

Hand the spec to the agent — it generates code that meets your defined contract.

4

Validate

Review, test, iterate. Feed results back to improve specs for the next cycle.

Power User Guidance

These Tips require good knowledge and time-invest, as they are conditional and / or come with trade-offs.

Think in Code

Prefer creating scripts to analyze files over feeding it to AI.

Consider CLIs vs. MCPs

CLI Tools can be more optimal in certain scenarios due to less static tokens.

Improve Shell Outputs

Shell outputs can be extremely long – use something like https://github.com/rtk-ai/rtk to reduce their outputs.

Run "/chronicle tip" regularly

Analyze your usage in Copilot CLI to find improvement areas.

Collapse Tool Calls

Using something like copilot-codeact-plugin can help collapse multiple tool calls into one.

Model Specific Context Optimization
